# 🔧 Refactoring Agent Workflow

Этот workflow запускает суб-агента для рефакторинга кода в репозитории.

## Перед запуском

Убедитесь, что вы понимаете какую часть кода нужно рефакторить:
- Конкретный файл или модуль
- Тип рефакторинга (структура, производительность, читаемость)

## Шаги выполнения

### 1. Анализ структуры кода

// turbo
```bash
# Найти большие файлы (>500 строк) - кандидаты на разбиение
find src -name "*.py" -exec wc -l {} + | sort -rn | head -20
```

### 2. Проверка сложности кода

// turbo
```bash
# Запуск анализа сложности через Ruff
python -m ruff check src --select=C901 --output-format=json 2>/dev/null || python -m ruff check src --select=C901
```

### 3. Поиск дублирующегося кода

// turbo
```bash
# Проверка на дублированный код через Ruff
python -m ruff check src --select=PLR --output-format=text
```

### 4. Анализ импортов и зависимостей

// turbo
```bash
# Проверка циклических импортов и порядка
python -m ruff check src --select=I --output-format=text
```

### 5. Форматирование кода

```bash
# Форматирование через Ruff (предварительный просмотр)
python -m ruff format src --diff
```

### 6. Автоисправление простых проблем

```bash
# Автоматическое исправление безопасных проблем
python -m ruff check src --fix --unsafe-fixes
```

### 7. Проверка типов после рефакторинга

// turbo
```bash
# MyPy для проверки типов
python -m mypy src --ignore-missing-imports --no-error-summary 2>&1 | head -50
```

### 8. Запуск тестов для проверки

// turbo
```bash
# Быстрые unit-тесты
python -m pytest tests/unit -x -q --tb=short 2>&1 | head -50
```

## Чек-лист рефакторинга

При рефакторинге проверяйте:

- [ ] **Разделение файлов**: Файлы > 500 строк разбиты на модули
- [ ] **Единственная ответственность**: Каждый класс/модуль имеет одну задачу
- [ ] **DRY**: Нет дублирующегося кода
- [ ] **Именование**: Ясные и описательные имена
- [ ] **Типы**: Аннотации типов добавлены
- [ ] **Docstrings**: Документация для публичных функций
- [ ] **Тесты**: Существующие тесты проходят
- [ ] **Импорты**: Правильный порядок импортов

## Паттерны рефакторинга

### Разбиение большого файла

```
old_module.py (800 строк)
↓
module/
├── __init__.py (экспорт публичного API)
├── core.py (основная логика)
├── helpers.py (вспомогательные функции)
├── types.py (типы и модели)
└── exceptions.py (исключения)
```

### Извлечение класса

Если класс слишком большой (>300 строк или >10 методов):
1. Выделить связанные методы в отдельный класс
2. Использовать композицию вместо наследования
3. Применить паттерн Strategy/Factory при необходимости

## После рефакторинга

1. Запустить полный набор тестов: `pytest tests/ -x`
2. Проверить linting: `ruff check src`
3. Проверить типы: `mypy src`
4. Обновить документацию при необходимости
