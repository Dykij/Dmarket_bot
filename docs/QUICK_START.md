## ⚠️ КРИТИЧЕСКИ ВАЖНО: Режим безопасности DRY_RUN

**ПРЕЖДЕ ЧЕМ НАЧАТЬ**, убедитесь, что понимаете разницу между режимами:

### 🔵 DRY_RUN=true (По умолчанию - БЕЗОПАСНО)

- ✅ Бот **НЕ** делает реальные сделки
- ✅ Все операции покупки/продажи **симулируются**
- ✅ Ваш баланс на DMarket **не изменяется**
- ✅ Идеально для тестирования и обучения
- ✅ Логи отмечены `[DRY-RUN]`

### 🔴 DRY_RUN=false (ОПАСНО - Реальные деньги!)

- ❌ Бот **ДЕЛАЕТ реальные сделки**
- ❌ Покупки и продажи **списывают/начисляют** реальные деньги
- ❌ Ошибки могут привести к **финансовым потерям**
- ⚠️ Используйте только после **48-72 часов тестирования** в DRY_RUN=true
- ⚠️ Логи отмечены `[LIVE]` с предупреждениями

**РЕКОМЕНДАЦИЯ**: Не переключайтесь на `DRY_RUN=false` до тех пор, пока:
1. Не протестируете бота минимум 48-72 часа в безопасном режиме
2. Не убедитесь, что бот работает корректно и стабильно
3. Не изучите все логи и не поймете поведение бота
4. Не прочитаете [docs/SECURITY.md](SECURITY.md)

```python
            print(f"Общая прибыль: ${status['total_profit']:.2f}")
            awAlgot asyncio.sleep(60)  # Проверка каждую минуту

asyncio.run(start_trading())
```

## 🎯 Как работают таргеты (Buy Orders)

### Что это?

**Таргет** - это заявка на покупку предмета по определенной цене. Когда продавец выставляет предмет, соответствующий вашему таргету, происходит автоматическая покупка.

### Преимущества

✅ **Покупка по вашей цене** - не переплачиваете
✅ **Автоматическое исполнение** - не нужно постоянно мониторить рынок
✅ **Приоритет** - ваша заявка выполняется первой
✅ **Гибкость** - можно указать специфические атрибуты (float, паттерн)

### Пример использования

```
1. Анализируете рынок и видите:
   AK-47 | Redline (FT) - текущая цена $8.50

2. Создаете таргет на $8.00 (на 5.9% ниже)

3. Когда кто-то выставит предмет за $8.00 или ниже,
   он автоматически купится на ваш аккаунт

4. Выставляете купленный предмет на продажу за $9.00

5. Прибыль: $9.00 * 0.93 (комиссия 7%) - $8.00 = $0.37 (4.6%)
```

### Стратегии использования таргетов

#### 1. Агрессивная стратегия

- Устанавливать цену на 3-5% ниже рынка
- Высокая вероятность исполнения
- Меньшая прибыль, но больше сделок

#### 2. Консервативная стратегия

- Устанавливать цену на 8-10% ниже рынка
- Низкая вероятность исполнения
- Высокая прибыль при исполнении

#### 3. Сбалансированная стратегия

- Устанавливать цену на 5-7% ниже рынка
- Средняя вероятность исполнения
- Оптимальная прибыль

## ⚠️ ВАЖНО: Проблема с кириллическими символами

### Частая ошибка при копировании команд

При копировании команд из документации или GitHub Copilot можете случайно вставить **русские буквы** вместо английских:

**❌ Неправильно:**
```bash
руtеst  # русские "р", "у", "е"
рip install  # русская "р"
руthоn -m src.mAlgon  # русские "р", "у", "о"
```

**✅ Правильно:**
```bash
pytest  # все буквы английские
pip install  # все буквы английские
python -m src.mAlgon  # все буквы английские
```

### Как избежать:

1. **Переключите клавиатуру на английский** перед вставкой:
   ```
   Win + Пробел → EN
   ```

2. **Проверяйте команды перед Enter:**
   - Выделите команду (Ctrl+A)
   - Увеличьте текст (Ctrl + колесо мыши)
   - Убедитесь, что все символы английские

3. **Используйте шрифт с четким различием:**
   - Установите **Cascadia Code** или **Fira Code**
   - Русская "с" и английская "c" будут выглядеть по-разному

## 🆘 Решение проблем

### Проблема: "Недостаточно средств"

**Решение**:

1. Проверьте баланс: `/balance`
2. Пополните счет DMarket
3. Уменьшите лимиты торговли

### Проблема: "Rate limit exceeded"

**Решение**:

1. Подождите 1-2 минуты
2. Уменьшите частоту сканирования
3. Увеличьте интервал между запросами в настSwarmках

### Проблема: "Таргеты не исполняются"

**Решение**:

1. Проверьте цену таргета (возможно, слишком низкая)
2. Увеличьте цену на 1-2%
3. Используйте "Умные таргеты" для автоматического расчета

### Проблема: "Автоторговля остановилась"

**Решение**:

1. Проверьте логи: `/logs`
2. Проверьте баланс
3. Проверьте лимиты
4. Перезапустите: `/arbitrage_start`

## 🔧 Устранение неполадок

### Проблема 1: "Ошибка валидации конфигурации"

**Симптомы**:
```text
❌ TELEGRAM_BOT_TOKEN not set or empty
```

**Решение**:
1. Убедитесь, что файл `.env` существует в корне проекта
2. Проверьте, что все обязательные переменные установлены
3. Убедитесь, что переменные не содержат лишних пробелов
4. Перезапустите терминал после изменения `.env`

```bash
# Проверить содержимое .env (Linux/macOS)
cat .env

# Проверить содержимое .env (Windows)
type .env

# Проверить переменные окружения
python -c "import os; print(os.getenv('TELEGRAM_BOT_TOKEN'))"
```

---

### Проблема 2: "Ошибка подключения к базе данных"

**Симптомы**:
```text
❌ Database connection fAlgoled
```

**Решение для SQLite**:
```bash
# Проверить путь к БД
python -c "from src.utils.config import Config; print(Config.load().database.url)"

# Создать директорию для БД, если не существует
mkdir -p data

# Повторно инициализировать БД
python scripts/init_db.py
```

**Решение для PostgreSQL**:
```bash
# Проверить доступность PostgreSQL
psql -h localhost -U your_user -l

# Создать базу данных
createdb dmarket_bot

# Проверить строку подключения
python -c "from src.utils.config import Config; print(Config.load().database.url)"
```

4. Сгенерируйте новые ключи при необходимости


3. Перезапустите бота

### Проблема 6: "Rate limit exceeded"

**Симптомы**:
```text
❌ DMarket API: Rate limit exceeded (429)
```

**Решение**:
- DMarket ограничивает ~30 запросов в минуту
- Подождите минуту перед повторной попыткой
- Проверьте настSwarmки rate limiting в коде:
  ```python
  # src/utils/rate_limiter.py
  max_calls=30,
  time_period=60  # seconds
  ```
