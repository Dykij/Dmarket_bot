# Финансовые инструменты рынка CS2 — DMarket Bot

## 1. Стакан ордеров (Order Book)

### Что это
Список всех активных покупок (bids) и продаж (asks) на конкретный предмет.

### Структура
```
ASK (Продажа):
  $15.00  x3  (3 предмета по $15)
  $14.50  x1
  $14.00  x5
--- SPREAD = $1.00 (6.7%) ---
BID (Покупка):
  $13.00  x2
  $12.50  x10
  $12.00  x1
```

### Метрики стакана
| Метрика | Формула | Что показывает |
|---------|---------|---------------|
| **Spread** | ask - bid | Ликвидность (меньше = лучше) |
| **Spread %** | (ask - bid) / ask × 100% | Маржа для арбитража |
| **OBI** (Order Book Imbalance) | (bid_vol - ask_vol) / (bid_vol + ask_vol) | Давление покупателей/продавцов |
| **OFI** (Order Flow Imbalance) | Δbid_vol - Δask_vol | Изменение давления |
| **Depth** | Сумма объёмов на ±10% от mid price | Глубина рынка |
| **Mid Price** | (best_bid + best_ask) / 2 | Справедливая цена |
| **VWAP** | Σ(price × volume) / Σ(volume) | Средневзвешенная цена |

### Применение в боте
- **OBI > 0.3** — покупатели доминируют, можно покупать
- **OBI < -0.3** — продавцы доминируют, ждать
- **Spread < 5%** — ликвидный рынок, безопасно торговать
- **Spread > 15%** — низкая ликвидность, высокий риск

## 2. VPIN (Volume-Synchronized Probability of Informed Trading)

### Что это
Метрика вероятности информированной торговли. Высокий VPIN = высокий риск манипуляций.

### Формула
```
VPIN = |V_buy - V_sell| / (V_buy + V_sell)
```
Где V_buy и V_sell определяются через Bulk Volume Classification (BVC).

### Применение
| VPIN | Уровень | Действие |
|------|---------|----------|
| < 0.3 | Низкий | Нормальная торговля |
| 0.3-0.6 | Средний | Повышенная осторожность |
| > 0.6 | Высокий | Приостановить покупки |

## 3. CVD (Cumulative Volume Delta)

### Что это
Кумулятивная разница между объёмом покупок и продаж. Показывает тренд давления.

### Формула
```
CVD += (buy_volume - sell_volume) за каждый период
```

### Применение
- **CVD растёт** — покупатели доминируют, бычий сигнал
- **CVD падает** — продавцы доминируют, медвежий сигнал
- **CVD дивергенция с ценой** — разворот тренда

## 4. Float Value Premium

### Что это
Надбавка к цене за низкий float (изношенность) скина. Чем ниже float, тем дороже.

### Шкала премий
| Float | Категория | Премия |
|-------|-----------|--------|
| 0.00-0.01 | Factory New (FN) | 25%+ |
| 0.01-0.03 | Minimal Wear (MW) | 15-20% |
| 0.03-0.07 | Field-Tested (FT) | 5-10% |
| 0.07-0.15 | Well-Worn (WW) | 0% |
| 0.15-0.38 | Battle-Scarred (BS) | 0% |

### Применение в боте
```python
if float_value < 0.01:
    multiplier = 1.25  # 25% premium
elif float_value < 0.03:
    multiplier = 1.20  # 20% premium
elif float_value < 0.07:
    multiplier = 1.08  # 8% premium
else:
    multiplier = 1.0   # no premium
```

## 6. Спред арбитраж (Spread Arbitrage)

### Что это
Покупка предмета на одной площадке и продажа на другой с прибылью.

### Формула прибыли
```
gross_profit = sell_price - buy_price
fee = sell_price × commission_rate
net_profit = gross_profit - fee
net_margin = net_profit / buy_price × 100%
```

### Пороговые значения
| Параметр | Значение | Описание |
|----------|----------|----------|
| min_spread | 5% | Минимальный спред для сделки |
| min_profit_usd | $0.50 | Минимальная прибыль в долларах |
| max_slippage | 2% | Максимальное проскальзывание |

## 7. Ликвидность (Liquidity Analysis)

### Что это
Способность быстро купить/продать предмет без существенного изменения цены.

### Метрики ликвидности
| Метрика | Формула | Интерпретация |
|---------|---------|---------------|
| **Volume** | Количество продаж за 24ч | Высокий = ликвидный |
| **Turnover** | volume × avg_price | Объём в долларах |
| **Days to Sell** | 1 / (volume / listings) | Сколько дней до продажи |
| **Slippage** | (exec_price - mid_price) / mid_price | Проскальзывание |

### Категории ликвидности
| Категория | Volume (24ч) | Days to Sell | Действие |
|-----------|-------------|-------------|----------|
| Highly liquid | >100 | <1 | Безопасно |
| Liquid | 20-100 | 1-3 | Осторожно |
| Illiquid | 5-20 | 3-7 | Высокий риск |
| Very illiquid | <5 | >7 | Не торговать |

## 8. Тренд-фильтр (Trend Guard)

### Что это
Фильтрация нисходящих трендов через анализ исторических цен.

### Методы определения тренда
| Метод | Формула | Сигнал |
|-------|---------|--------|
| **SMA crossover** | SMA(7) vs SMA(30) | SMA(7) > SMA(30) = бычий |
| **Price momentum** | (price_now - price_7d_ago) / price_7d_ago | >0 = бычий |
| **Consecutive drops** | Подряд идущие падения | >3 = нисходящий |
| **Crash detection** | Падение >20% за 24ч | = крах, стоп |

### Применение
```python
if is_crashing(consecutive_drops=3):
    SKIP_ITEM  # Не покупать падающий предмет
if price_momentum < -0.10:
    SKIP_ITEM  # Нисходящий тренд
```

## 10. Kelly Criterion (Position Sizing)

### Что это
Математическая формула для оптимального размера позиции.

### Формула
```
f* = (p × b - q) / b
```
Где:
- f* = оптимальная доля капитала
- p = вероятность выигрыша
- q = вероятность проигрыша (1 - p)
- b = соотношение выигрыша/проигрыша

### Half Kelly (применяется в боте)
```python
full_kelly = (win_rate * avg_win - (1 - win_rate) * avg_loss) / avg_win
position_size = full_kelly * 0.5  # 50% от Kelly
```

### Преимущества Half Kelly
- Снижает просадку на ~50%
- Сохраняет ~85% от роста полного Kelly
- Более стабильная кривая капитала

## 11. Drawdown Management

### Типы просадки
| Тип | Формула | Действие |
|-----|---------|----------|
| **Daily loss** | realized_pnl за день | Лимит $10/день |
| **Max drawdown** | (peak - current) / peak × 100% | >15% = стоп |
| **Soft halt** | drawdown > 5% | Уменьшить позиции вдвое |
| **Hard halt** | drawdown > 15% | Полная остановка |

### Состояния
```
NORMAL → (DD > 5%) → SOFT_HALT → (DD > 15%) → HARD_HALT
                                              ↓
                              (DD < 5%) → NORMAL
```

## 12. Event Shield

### Что это
Повышение минимального спреда во время крупных CS2 событий.

### События
| Событие | Влияние | Действие |
|---------|---------|----------|
| CS2 Major | Рост цен на 20-50% | Увеличить min_spread до 15% |
| Новый кейс | Волатильность | Увеличить min_spread до 10% |
| Обновление игры | Непредсказуемость | Приостановить торговлю |
| Распродажа Steam | Падение цен | Уменьшить позиции |
